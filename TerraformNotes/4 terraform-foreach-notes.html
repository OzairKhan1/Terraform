for_each Loop

All variations · Examples · Output · Best Practices

01

What is for_each?

for_each is a meta-argument in Terraform that lets you create multiple instances of a resource or module from a single block — by looping over a map or set.

⚡

Golden Rule: for_each only accepts a Map or a Set. It will error on a plain list. You must convert using toset() or an expression.

✅ Valid
# Map
for_each = var.tags

# Set
for_each = toset(["a", "b"])
❌ Invalid
# Plain list — will ERROR
for_each = ["a", "b"]

# List variable — will ERROR
for_each = var.my_list
02

Option 1 — List → toset()

When you have a plain list of strings or numbers, wrap it with toset() to convert it into a set.

variables.tf
variable "environments" {
  type    = list(string)
  default = ["dev", "staging", "prod"]
}
main.tf
resource "aws_s3_bucket" "env_buckets" {
  for_each = toset(var.environments)  # convert list → set

  bucket = "myapp-${each.value}-bucket"   # each.value = "dev" / "staging" / "prod"

  tags = {
    Name = "${each.value}"
  }
}
Resources Created (3 S3 Buckets)
aws_s3_bucket.env_buckets["dev"]      → myapp-dev-bucket
aws_s3_bucket.env_buckets["staging"]  → myapp-staging-bucket
aws_s3_bucket.env_buckets["prod"]     → myapp-prod-bucket
💡

With toset(), you only have each.value — there is no each.key (or each.key = each.value). Duplicates are automatically removed.

03

Option 2 — Map directly (each.key / each.value)

When your variable is already a map, pass it directly. You get both each.key and each.value.

variables.tf
variable "instance_types" {
  type = map(string)
  default = {
    dev     = "t2.micro"
    staging = "t2.medium"
    prod    = "t3.large"
  }
}
main.tf
resource "aws_instance" "servers" {
  for_each      = var.instance_types   # map used directly — no conversion

  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = each.value           # "t2.micro" / "t2.medium" / "t3.large"

  tags = {
    Name        = "${each.key}-server"  # "dev" / "staging" / "prod"
    Environment = each.key
  }
}
Resources Created (3 EC2 Instances)
aws_instance.servers["dev"]     → ami-xxx  t2.micro   Name=dev-server
aws_instance.servers["staging"] → ami-xxx  t2.medium  Name=staging-server
aws_instance.servers["prod"]    → ami-xxx  t3.large   Name=prod-server
04

Option 3 — List of Objects → for expression

When you have a list of objects (complex types), you must convert them to a map using a for expression first.

variables.tf
variable "users" {
  type = list(object({
    name = string
    role = string
  }))
  default = [
    { name = "Ali",  role = "admin" },
    { name = "Sara", role = "dev"   },
    { name = "John", role = "ops"   }
  ]
}
main.tf
resource "aws_iam_user" "team" {
  # Convert list of objects → map using for expression
  # Syntax: { for item in list : item.unique_key => item }
  for_each = { for u in var.users : u.name => u }

  name = each.key           # "Ali" / "Sara" / "John"

  tags = {
    Role = each.value.role   # "admin" / "dev" / "ops"
    Name = each.key
  }
}
Resources Created (3 IAM Users)
aws_iam_user.team["Ali"]  → name=Ali  Role=admin
aws_iam_user.team["Sara"] → name=Sara Role=dev
aws_iam_user.team["John"] → name=John Role=ops
💡

The key you choose (u.name here) must be unique across all items — Terraform uses it to track the resource in state.

💬

Common Confusion — each.key vs object key

Coming from Python, it's natural to think each.key refers to the field name inside the object (like "name" or "role"). But in Terraform it means something completely different.

In Python — key is the field name inside the dict

Python
user = { "name": "Ali", "role": "admin" }
#          ↑ key     ↑ value   ↑ key     ↑ value

for k, v in user.items():
    print(k, v)
# name  Ali
# role  admin

In Terraform — each.key is the map identifier YOU chose

When you write the for expression, you are building a brand new map and you decide what becomes the key:

Terraform — for expression breakdown
for_each = { for u in var.users : u.name => u }
#                                      ↑             ↑
#                              "Ali" becomes       whole object
#                               the map KEY        is the VALUE

Terraform internally builds this map before looping:

Map Terraform builds internally
{
  "Ali"  = { name = "Ali",  role = "admin" }   # key="Ali"  value=whole object
  "Sara" = { name = "Sara", role = "dev"   }   # key="Sara" value=whole object
  "John" = { name = "John", role = "ops"   }   # key="John" value=whole object
}

Now each.key and each.value make sense:

Expression What it refers to Example value
each.key The identifier you picked (u.name) "Ali"
each.value The whole object { name="Ali", role="admin" }
each.value.role A field inside the object "admin"
each.value.name Another field inside the object "Ali"
🧠

The key mental shift from Python:

Python → key = field name inside the object ("name", "role")

Terraform → each.key = the unique identifier you chose to represent the whole object in the map ("Ali", "Sara")

You could choose a different key too

name as key
for u in var.users : u.name => u
# key = "Ali", "Sara", "John"
role as key (if unique)
for u in var.users : u.role => u
# key = "admin", "dev", "ops"
⚠️

Whatever you pick as key must be unique across all items — Terraform uses it to track the resource in the state file. Duplicate keys will cause an error.

05

Option 4 — dynamic block with for_each

Used inside a resource to dynamically generate nested blocks (like ingress/egress rules). The content {} block defines what each iteration generates.

variables.tf
variable "allowed_ports" {
  type    = list(number)
  default = [80, 443, 22]
}

variable "allowed_egress_ports" {
  type    = list(number)
  default = [80, 443]
}
main.tf
resource "aws_security_group" "web_sg" {
  name = "${var.environment}-web-sg"

  # Dynamic ingress — loop through port list
  dynamic "ingress" {
    for_each = toset(var.allowed_ports)   # list → set
    content {
      from_port   = ingress.value           # iterator = block name by default
      to_port     = ingress.value
      protocol    = "tcp"
      cidr_blocks = ["0.0.0.0/0"]
    }
  }

  # Dynamic egress — with custom iterator name
  dynamic "egress" {
    for_each = toset(var.allowed_egress_ports)
    iterator = port                           # custom iterator name
    content {
      from_port   = port.value               # use "port" instead of "egress"
      to_port     = port.value
      protocol    = "tcp"
      cidr_blocks = ["0.0.0.0/0"]
    }
  }
}
Rules Generated
Ingress Rules:  port 80  (tcp) ← 0.0.0.0/0
                port 443 (tcp) ← 0.0.0.0/0
                port 22  (tcp) ← 0.0.0.0/0

Egress Rules:   port 80  (tcp) → 0.0.0.0/0
                port 443 (tcp) → 0.0.0.0/0
★

Quick Reference Summary

Situation Syntax Access
Plain list of strings for_each = toset(var.list) each.value only
Map variable for_each = var.map each.key + each.value
List of objects for_each = { for x in var.list : x.id => x } each.key + each.value.field
Dynamic nested blocks dynamic "block" { for_each = ... content {} } block.value (or iterator.value)
Custom iterator name iterator = port inside dynamic port.key / port.value
⚠️

Never use plain list directly — always convert to set or map first. And always use a stable, unique key for maps — Terraform tracks resources by key in state file.
